To flash your device, you need to obtain the flashable image archive
either from your build output or by downloading a prebuilt package from
an artifactory. These packages follow the naming format
`<IMAGE>-<MACHINE>.rootfs.qcomflash.tar.gz`. The image is typically provided
as a compressed archive and must be unpacked before use.

- Extract the archive.The following examples use
  `qcom-multimedia-proprietary-image` as the image name

  <CodeGroup>

  ```bash IQ-9075-EVK
  tar -xvf qcom-multimedia-proprietary-image-iq-9075-evk.rootfs.qcomflash.tar.gz
  ```

  ```bash IQ-8275-EVK
  tar -xvf qcom-multimedia-proprietary-image-iq-8275-evk.rootfs.qcomflash.tar.gz
  ```

  ```bash IQ-615-EVK
  tar -xvf qcom-multimedia-proprietary-image-iq-615-evk.rootfs.qcomflash.tar.gz
  ```

  ```bash QCS6490
  tar -xvf qcom-multimedia-proprietary-image-rb3gen2-core-kit.rootfs.qcomflash.tar.gz
  ```

  </CodeGroup>

- Navigate to the unpacked directory. All subsequent instructions are expected to be
  run from this directory unless stated otherwise.

  <CodeGroup>

  ```bash IQ-9075-EVK
  cd qcom-multimedia-proprietary-image-iq-9075-evk.rootfs.qcomflash
  ```

  ```bash IQ-8275-EVK
  cd qcom-multimedia-proprietary-image-iq-8275-evk.rootfs.qcomflash
  ```

  ```bash IQ-615-EVK
  cd qcom-multimedia-proprietary-image-iq-615-evk.rootfs.qcomflash
  ```

  ```bash QCS6490
  cd qcom-multimedia-proprietary-image-rb3gen2-core-kit.rootfs.qcomflash
  ```

  </CodeGroup>
