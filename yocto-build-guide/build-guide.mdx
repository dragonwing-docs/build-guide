## Introduction

This document provides guidance on how to build a Qualcomm-based software image
from scratch using the Yocto Project. It is intended for developers who need
to customize and generate software images for specific hardware platforms.

The meta-qcom layer, provided by Qualcomm, serves as the primary entry point
for this process. It integrates the required configurations, recipes, and
dependencies needed to build Yocto-based images targeted for Qualcomm platforms.

As part of this guide, you'll
- Verify if your host system meets the build requirements - [Host System Requirements](#host-system-requirements)
- Setup your host machine and install the required dependencies - [Host Setup](#host-setup)
- Clone a validated release snapshot of meta-qcom - [Build Instructions](#build-instructions)
- Use the kas tool to build the image based on chosen machine and distribution - [Build Instructions](#build-instructions)
- Verify the build outputs by checking the generated artifacts - [Build Instructions](#build-instructions)
